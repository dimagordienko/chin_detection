# Chin Detection (YOLO Pose Fine-Tuning)

Этот проект демонстрирует самый легкий и быстрый процесс дообучения (fine-tuning) модели **YOLO Pose** для задачи детекции подбородка (chin detection) с использованием ключевых точек.


#  1. Подготовка модели

Скачайте предобученные веса YOLO Pose:

```
yolo11s-pose.pt
```

Это базовая модель, которую мы будем дообучать под нашу задачу.

# 3. Разметка данных

Разметка выполняется через CVAT или аналогичные инструменты.

Каждая строка в `.txt` файле имеет формат:

```
class x_center y_center width height kpt_x kpt_y visibility
```

### Пример:

```
0 0.493021 0.371556 0.05 0.05 0.493021 0.371556 2
```

---

# 4. Расшифровка разметки

| Параметр             | Описание                     |
| -------------------- | ---------------------------- |
| `0`                  | класс объекта (chin)         |
| `x_center, y_center` | центр bounding box           |
| `w, h`               | ширина и высота bounding box |
| `kpt_x, kpt_y`       | координаты ключевой точки    |
| `visibility`         | видимость точки              |
| `0`                  | не размечена                 |
| `1`                  | есть, но не видна            |
| `2`                  | видна                        |

---

# 5. Конфигурация `data.yaml`

Пример настройки:

```yaml
path: dataset

train: images/train
val: images/val
test: images/test

names:
  0: chin

kpt_shape: [1, 3]

flip_idx: [0]
```

---

### Пояснения:

* `names` — классы объектов (в моём случае только `chin`)
* `kpt_shape: [1, 3]` означает:

  * 1 ключевая точка
  * каждая точка содержит: `(x, y, visibility)`
* `flip_idx` — правило зеркального отражения ключевых точек

---

### Пример flip_idx

Если бы было несколько точек:

```
nose (0)
left_eye (1)
right_eye (2)
```

Тогда:

```yaml
flip_idx: [0, 2, 1]
```

# 6. Обучение модели

Установите библиотеки:

```bash
pip install ultralytics
pip install mlflow
```

Откройте ноутбук:

```
train_model.ipynb
```

Проверьте пути к датасету и модели перед запуском.

---

# 7. Что такое epochs

**Epochs** — это количество полных проходов модели по всему датасету.

* мало данных → больше epochs
* много данных → меньше epochs

---

# 8. Результаты обучения

Во время обучения создаются папки с логами:

```
runs/pose/train/
```

Там можно найти:

* графики обучения
* примеры предсказаний (batch images)
* метрики

---

# 9. Сохранённая модель

После обучения лучшая модель сохраняется здесь:

```
runs/pose/train/weights/best.pt
```

Это финальные веса, которые можно использовать в продакшене.

---

# 10. Использование модели

Пример инференса:

```python
from ultralytics import YOLO

model = YOLO("runs/pose/train/weights/best.pt")

results = model("image.jpg")
```
